The architecture is the navigation

Data flows through six stages. Scroll to fly the network from raw extraction to the product surface.

01
Ingest

12 headless extractors, on a schedule

Playwright drives real browser sessions to pull operational reports the vendor exposes no API for — event-driven waits, never blind sleeps, so extraction stays fast and resilient when pages change.

PythonPlaywrightHeadless browser
02
Transform

18 cleaners turn messy exports into typed facts

Each raw export is normalized, typed, and deduplicated into clean fact tables — the boring, essential layer that makes everything downstream trustworthy.

PythonPandasData modeling
03
Warehouse

An append-only snapshot model that time-travels

Every fact is keyed by snapshot_date in PostgreSQL (Neon). Nothing is overwritten — so the business can replay any past day and see exactly what was true then. Trends and history come for free.

PostgreSQLNeonSnapshot warehouse
04
Serve

A FastAPI read layer, pooled and cached

15 routers behind a process-wide connection pool and TTL caching, so the dashboard reads are fast and the warehouse isn't hammered. Clean separation between data and product.

FastAPIConnection poolTTL cache
05
Product

The operations dashboard people actually run

A React / Vite / Tailwind app that turns the warehouse into a working surface: the screen ownership opens first each morning to decide what to do next.

ReactViteTailwind CSS
06
Orchestrate

Four scheduled jobs keep it unattended

A daily full extract at 06:00, an end-of-day fast refresh at 17:00, an hourly near-realtime sync, and a minute-level SMS worker — running on their own, no one babysitting the pipeline.

Task SchedulerEvent-drivenZero manual steps

What the product actually does

Every surface below is live in the dashboard — real domain logic, not mockups.

// actions

Prioritized actions feed

Ranks every operational signal into a single "do this now" workstream, so a manager starts with the highest-leverage task instead of a wall of reports.

// risk

Tenant risk scoring

Deterministic risk tiers computed from delinquency, late-payment history, lease expirations, and violations — explainable, not a black box.

// rules

Delinquency & eviction rules engine

Domain logic modeled precisely: grace through day 5, delinquent from day 6, eviction-eligible from day 11 at a balance ≥ rent. Encoded, not guessed.

// leasing

Leasing pipeline

Vacancy, applicant funnel, and lease-expiration risk in one view — so nothing sits empty or lapses unnoticed.

// maintenance

Maintenance tracker

Open work orders by age and criticality, surfaced before they become emergencies or tenant complaints.

// finance

Financials

Cash-basis P&L, collection rates, and delinquency trends read straight from the snapshot warehouse.

// ai

Privacy-aware AI layer

OpenRouter summarizes applicant screening and suggests tenant actions — operating only on anonymized metrics, private by design.

// sms

Outbound SMS worker

A message queue drained by a dedicated Playwright worker for real two-way tenant communication, wired into the same data.
