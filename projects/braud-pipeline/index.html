Overview

This engagement produced two live production deliverables for a Monroe, LA property management company — engineered entirely without native Buildium API access.

The Braud Pipeline is the data layer: nightly Playwright + Prefect automation that extracts 17–19 operational Buildium reports, transforms and validates them, and loads structured records into PostgreSQL — zero manual steps after deployment.

Braud Operations Dashboard is the analytics layer: an owner-facing UI that reads from that database and surfaces Morning Briefing, Executive KPIs, Decision Inbox, and Occupancy modules — turning pipeline output into daily operational decisions around delinquency, vacancy, maintenance SLAs, and lease renewals.

Deliverable 1 — The Braud Pipeline

Three-stage ETL automation — from raw Buildium exports to a stable PostgreSQL schema.

01

Extract

Playwright headless automation logs in, navigates each report, and downloads files — no API required.

02

Transform

Python normalizes report formats, deduplicates records, and validates data before load.

03

Load

Structured data lands in PostgreSQL — stable schema supporting occupancy, delinquency, and maintenance analytics.

Deliverable 2 — Braud Operations Dashboard

The analytics front-end reads pipeline output and gives Braud ownership a single place to start each day. A sanitized standalone demo is available for portfolio review — fictional data only.

Pipeline architecture — Buildium to PostgreSQL

Owner-facing analytics dashboard

The Problem

Buildium's Growth plan provides no native API access. All reporting was done manually — someone had to log in, navigate to each report, download it, and process it by hand. 17–19 reports. Daily. Ownership had no single view of delinquency, vacancy, or maintenance pressure without someone assembling it first.

The Solution

Pipeline: Playwright headless automation handles login, navigation, and extraction. Prefect orchestrates the nightly schedule and handles retries. PostgreSQL stores structured clean data — 17+ reports with zero manual steps.

Dashboard: A custom UI layer surfaces morning briefings, executive KPIs, a Decision Inbox with drafted actions, and occupancy snapshots — giving ownership a format they can act on without touching a spreadsheet.
