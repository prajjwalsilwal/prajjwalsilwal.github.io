Overview

End-to-end automation system built with Playwright and Prefect that extracts and processes 17–19 operational reports from Buildium, loads clean data into a PostgreSQL backend, and powers a live owner-facing analytics dashboard — engineered entirely without native API access.

The dashboard layer translates pipeline output into daily operational decisions: delinquency tracking, vacancy pressure, maintenance SLAs, lease renewals, and a shared Decision Inbox for ownership workflows.

Pipeline Architecture

Four stages — from raw Buildium exports to actionable dashboard views.

01

Extract

Playwright headless automation logs in, navigates each report, and downloads files — no API required.

02

Transform

Python normalizes report formats, deduplicates records, and validates data before load.

03

Load

Structured data lands in PostgreSQL — stable schema supporting occupancy, delinquency, and maintenance analytics.

04

Surface

Dashboard UI reads from the database and presents KPIs, briefing tables, and workflow queues to ownership.

Dashboard Layer

The analytics front-end built on top of the pipeline gives Braud ownership a single place to start each day — fictional demo available for portfolio review (sanitized data only).

The Problem

Buildium's Growth plan provides no native API access. All reporting was done manually — someone had to log in, navigate to each report, download it, and process it by hand. 17–19 reports. Daily. Ownership had no single view of delinquency, vacancy, or maintenance pressure without someone assembling it first.

The Solution

Playwright headless automation handles login, navigation, and extraction. Prefect orchestrates the schedule and handles retries. PostgreSQL stores structured clean data. The dashboard layer surfaces morning briefings, executive KPIs, a Decision Inbox with drafted actions, and occupancy snapshots — giving ownership a format they can actually act on without touching a spreadsheet.
